# First-Wave Outreach Targets

Who to contact first, why, and what counts as a real signal.

## 1. Who

| Category | Profile |
|----------|---------|
| **Claude Code / OpenClaw skill builders** | Already build and install agent skills. Lowest friction to try Chorus as a skill. |
| **Multi-language agent developers** | Building agents that serve users across languages. Chorus solves their "translate vs. adapt" gap. |
| **Agent-to-agent tooling builders** | Working on agent orchestration, A2A protocols, or multi-agent systems. Chorus is a concrete interop format they can evaluate. |
| **Cross-locale product teams** | Shipping agents to markets where cultural adaptation matters (CN/JP/KR/LATAM). They already feel the pain. |

## 2. What to Send

Same package for all categories:

- **Install**: `npx @chorus-protocol/skill init --target openclaw` — [npm](https://www.npmjs.com/package/@chorus-protocol/skill)
- **Try it**: [quick-trial.md](quick-trial.md) — install, generate an envelope, see it work in 5-10 min
- **All install paths**: [openclaw-install.md](openclaw-install.md) — OpenClaw, Claude Code, local, or direct copy
- **One-line pitch**: "Chorus is an open envelope format that lets agents send messages across languages and cultures — not just translate, but adapt."

## 3. What Counts as a Signal

| Signal | Value |
|--------|-------|
| Ran `npx @chorus-protocol/skill init --target openclaw` | They tried |
| Generated a valid envelope (with or without a server) | They understood the protocol |
| Filed an issue or asked a question about integration | They hit real friction — actionable feedback |
| Reported a doc gap or confusing step | Direct improvement input |
| Built something on top (even a prototype) | Adoption |

**Not a signal**: likes, retweets, "looks cool", or forwarding without trying.

## 4. Channels

Reach individuals, not audiences. Direct messages or small group threads — not broadcast posts.

- People you already know who build multi-agent or multilingual systems
- Claude Code skill community (if a channel exists)
- Small Discord/Slack groups focused on agent tooling
- Direct GitHub issues or discussions on related projects
