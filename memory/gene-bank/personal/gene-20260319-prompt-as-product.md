# 提示词是产品，不是技术细节

**类型**: Heuristic
**来源**: Chorus Protocol Phase 2 — 2026-03-19
**场景**: AI 产品的体验设计

## 内容

一行提示词的改变可以产生产品级别的体验质变。

Chorus Protocol 案例：
- 改前："请用目标文化最自然的方式表达这段消息。只输出适配文本。"
  → 输出："お元気ですか？"（隐形翻译机）

- 改后："你是用户的私人跨文化助手。像懂两种文化的朋友传话。"
  → 输出："相手は挨拶として送ってきたつもりです。中国では古くから使われる挨拶で…"（有人格的文化朋友）

零代码改动，零架构变更，纯提示词调整——但产品定位从"翻译工具"变成了"文化伙伴"。

启发：在 AI 产品中，提示词 = 产品设计。应该像对待 UI 一样对待提示词：迭代、A/B 测试、用户反馈驱动。
