# SHA-256 for System-Generated API Keys, bcrypt for Passwords

## Action

Use SHA-256 (or similar fast hash) for storing system-generated tokens/API keys. Reserve bcrypt/scrypt/argon2 for user-chosen passwords which may have low entropy. The key insight: dictionary attacks are irrelevant when the secret space is 2^128.

## Evidence

- 2026-03-22: Chorus hub generates `ca_${uuid.replace(/-/g, '')}` keys (128-bit entropy). SHA-256 hash stored in SQLite, plaintext returned once at registration. Auth lookup hashes incoming Bearer token and compares. Zero performance overhead vs bcrypt's intentional slowness.
