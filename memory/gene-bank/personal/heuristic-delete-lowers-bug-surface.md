# 删代码 = 删 bug 面——净删除行数是重构质量指标

**类型**: Heuristic
**来源**: Chorus Phase 6 · 2026-03-20
**场景**: 评估重构是否成功

## 内容

当重构（非新功能）的 diff 是净负数（删多于增），通常意味着：
1. 发现了不必要的抽象层
2. 删除了不需要的中间步骤
3. 简化了数据流

Chorus Phase 6 对齐重构：18 files, -868 +393 = **net -475 lines**。
同时 141 tests 全绿、覆盖率 82.82%、tsc 零错误。

**验证方式**: 如果净删除 > 30% 的变更行数，且测试不变红，这是一次高质量重构。
Phase 6 的删除比例: 868/(868+393) = 68.8%。
