# 一句话类比法：用已知成功案例锚定新协议的价值

**类型**: Heuristic
**来源**: Chorus Phase 4 Zed/ACP 文章分析 — 2026-03-19
**场景**: 向外部解释一个新协议/产品的价值定位时

## 内容

ACP 用"AI 时代的 LSP"一句话讲清了整个协议的价值。这个技巧有三层机制：

### 1. 共识锚定（Problem Anchoring）

听众必须**亲身经历过**类比来源的痛点。LSP 之前每个开发者都配过语言支持，这是体感记忆，不需要解释。当 ACP 说"同样的问题"，听众已经在点头了。

### 2. 解法迁移（Solution Pattern Transfer）

听众自动完成推导：
- "哦，每个 Agent 只需实现一次协议"
- "哦，换编辑器不影响我用的 Agent"
- "哦，新 Agent 出来所有编辑器自动能用"

**你什么都没说，但他什么都懂了。**

### 3. 可信度借用（Credibility Transfer）

LSP 是已被证明的成功案例。类比隐含潜台词："这不是实验，是在走一条已验证的路。"

## Chorus 的类比验证

"Chorus 是 AI Agent 之间的 SMTP"

| 维度 | 要求 | SMTP 满足？ |
|------|------|:---------:|
| 听众有体感 | 每个人都发过邮件 | ✅ |
| 解法能迁移 | 客户端-协议-服务器 | ✅ |
| 成功案例 | SMTP 统一了邮件生态 | ✅ |
| 一句话可转述 | "AI Agent 的邮局" | ✅ |

SMTP 还多一层优势：天然包含"内容无关性"——邮局不管你写的是中文还是英文，就像 Chorus Router 不管你是什么文化。这正是 Chorus 的 receiver-side adaptation 设计理念。
