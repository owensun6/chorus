## 功能详情

### F1: 流式输出

| 编号 | 功能描述 | 操作类型 |
|------|---------|---------|
| F1.1 | 发送端提取语义时，结果逐段返回，不等待完整响应 | 接口调用 |
| F1.2 | 接收端文化适配时，适配文本逐字生成并实时推送给用户 | 接口调用 |
| F1.3 | 路由服务器收到目标 Agent 的流式响应后，逐块转发给发送方，不缓存完整响应 | 接口调用 |
| F1.4 | 命令行界面逐字显示接收到的适配文本，而非等待完整响应后一次性打印 | 界面操作 |

### F2: Web Demo UI

| 编号 | 功能描述 | 操作类型 |
|------|---------|---------|
| F2.1 | 左右分栏界面，左栏代表 zh-CN Agent 视角，右栏代表 ja Agent 视角，各自有输入框和消息列表 | 界面操作 |
| F2.2 | 每条消息下方可展开查看信封元数据：语义意图、文化背景说明、意图类型、正式度、情感基调 | 界面操作 |
| F2.3 | 运行一条命令即可同时启动路由服务器、两个 Agent、Web 服务器，浏览器自动打开 demo 页面 | 界面操作 |
| F2.4 | 消息发送后，对方栏位实时显示适配文本逐字出现的过程（依赖 F1 流式输出） | 界面操作 |

### F3: 多轮对话上下文

| 编号 | 功能描述 | 操作类型 |
|------|---------|---------|
| F3.1 | 每个 Agent 在内存中维护与每个对话方的消息历史，包含原文和适配结果 | 增删改查 |
| F3.2 | 文化适配时，将最近 N 轮（默认 10）对话历史注入提示词，帮助理解上下文 | 接口调用 |
| F3.3 | 信封升级为 v0.3，新增对话标识和轮次编号字段，用于追踪同一对话中的消息顺序 | 接口调用 |
