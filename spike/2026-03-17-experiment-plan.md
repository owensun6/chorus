# Chorus Protocol — 完整实验计划

> 日期: 2026-03-17 | 执行时间: 今晚过夜运行 | Provider: MiniMax M2.5 + Dashscope Kimi-K2.5

---

## 一、实验目标

1. **受控实验**: 多语言对 × 三组对比 → 验证 A-05/A-08 在不同文化距离下是否成立
2. **对话模拟**: 多语言对 × 真实对话 → 展示 Chorus 协议的实际效果
3. **产出**: 每个实验有截图 + JSON 报告 + 分析摘要

## 三、受控实验计划

### 3.1 测试语料

每个语言对 40 条（20 禁忌 + 20 俚语），双向各半。

| 语言对 | 条数 | 方向 |
|--------|------|------|
| zh-CN ↔ ja | 40 | 20 zh→ja + 20 ja→zh |
| zh-CN ↔ ko | 40 | 20 zh→ko + 20 ko→zh |
| zh-CN ↔ en-US | 40 | 20 zh→en + 20 en→zh |
| ja ↔ en-US | 40 | 20 ja→en + 20 en→ja |
| zh-CN ↔ ar | 40 | 20 zh→ar + 20 ar→zh |
| zh-CN ↔ fr | 40 | 20 zh→fr + 20 fr→zh |
| **合计** | **240** | |

### 3.2 实验矩阵

每条语料 × 3 组 × 2 provider × (1 翻译 + 1 评分) = 每条 12 次 LLM 调用

| 指标 | 数值 |
|------|------|
| 总语料 | 240 条 |
| LLM 调用 | 240 × 12 = 2,880 次 |
| 预估时间 | ~3-4 小时（每次调用 ~4 秒） |
| 预估 token | ~500 万 input + ~200 万 output |

### 3.3 产出物（每个语言对）

```
spike/results/
├── controlled/
│   ├── zh-ja/
│   │   ├── report.json              # 完整数据
│   │   ├── summary.md               # 分析摘要
│   │   └── screenshot.png           # 面板截图
│   ├── zh-ko/
│   │   ├── ...
│   ├── zh-en/
│   ├── ja-en/
│   ├── zh-ar/
│   └── zh-fr/
└── overview-report.md               # 六组汇总对比
```

### 3.4 分析维度

每个语言对的 summary.md 包含：

1. **假设验证**: A-05/A-08 在该语言对是否成立
2. **分数对比表**: A/B/C 三组 × 三维度 × 两个 provider
3. **最佳/最差案例**: 提升最大的 3 条 + 提升最小的 3 条，附原文和翻译对比
4. **Provider 差异**: MiniMax vs Dashscope 在该语言对的表现差异
5. **关键发现**: 该语言对特有的文化适配挑战

汇总 overview-report.md 包含：

1. **六组横向对比**: 文化距离 vs 适配提升幅度（是否距离越远提升越大？）
2. **Provider 总评**: MiniMax vs Dashscope 各自擅长的语言对
3. **协议设计建议**: 哪些可选字段在哪些语言对中最有价值

## 四、对话模拟计划

### 4.1 对话角色设定

| 语言对 | 角色 A | 角色 B | 对话场景 |
|--------|--------|--------|---------|
| zh ↔ ja | Alice 上海产品经理 30F | Bob 东京工程师 32M | 初次商务社交，聊工作和生活 |
| zh ↔ ko | 小明 北京大学生 22M | 지수 首尔设计师 25F | 网友初次聊天，聊流行文化 |
| zh ↔ en | 李华 深圳创业者 35M | Sarah 硅谷 PM 33F | 跨国项目合作初次沟通 |

- 轮数: 8 轮（A→B→A→B... 共 8 个来回）
- Provider 分配: 角色 A 用 MiniMax，角色 B 用 Dashscope
- 自动进行，无人工干预

### 4.3 产出物（每个语言对）

│   │   ├── transcript.json           # 完整对话记录（含信封）

### 4.4 LLM 调用量

## 七、风险与降级

| 风险 | 降级方案 |
|------|---------|
| API 限流 | 调用间增加 1 秒延迟；若持续限流，减少到 3 个语言对 |
| 阿拉伯语/韩语质量差 | 记录并分析，不跳过——质量差本身也是有价值的数据 |
| 某个 provider 中途挂掉 | 另一个继续跑；挂掉的 provider 待恢复后补跑 |
| Token 不够 | 优先跑 zh-ja / zh-en / zh-ar 三组（文化距离梯度最大） |
